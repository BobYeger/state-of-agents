# The Bitter Lesson

## Summary

- Argues that the most durable progress in AI comes from general methods that scale with computation, especially search and learning.
- Uses chess, Go, speech recognition, and computer vision as examples where hand-built domain knowledge helped in the short run but was eventually outperformed by scalable computation-heavy methods.
- The lesson is relevant to agent systems because agent architectures should be judged by whether they preserve scalable search, learning, tool use, evaluation, and adaptation rather than encoding brittle human assumptions.

## Claims

- Durable AI progress tends to come from scalable general methods rather than hand-coded domain knowledge.
- Agent architecture should avoid overfitting to current task intuitions when a more general search, learning, or optimization loop can improve with scale.
- The lesson is not "ignore structure"; it is "prefer structure that lets search and learning exploit more compute, data, feedback, and experience."
