# Web Agents Should Adopt the Plan-Then-Execute Paradigm

## Summary

- Argues that ReAct-style web agents are vulnerable because untrusted web content can steer control flow during execution.
- Makes the security case for committing to a task-specific plan before observing runtime web content.
- Important bridge between deliberative control, computer-use agents, and prompt-injection defense.
