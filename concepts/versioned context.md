# Versioned Context

Versioned context treats prompts, policies, skills, memories, examples, and project instructions as managed artifacts with history, review, ownership, and deployment stages.

This is the operational version of context engineering. The context is no longer a single system prompt; it is a repository-like layer that can be edited by humans or agents, tested, rolled back, and loaded just in time.
