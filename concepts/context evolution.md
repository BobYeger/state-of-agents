# Context Evolution

Context evolution treats the active context as a changing working memory that can be expanded, refined, retrieved into, reasoned over, or deliberately kept small.

The key move is deciding when to retrieve new evidence and when to reason with what is already available. This connects context engineering with orchestration: the harness or a controller agent decides how the working context should change over time.
